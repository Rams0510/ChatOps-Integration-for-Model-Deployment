# ML Deploy — Phase 1: Containerize Your Model

Complete, runnable code for Phase 1 of the ML deploy pipeline.

## Project structure

```
ml-deploy/
├── train_model.py       ← run once to produce model.pkl
├── requirements.txt
├── Dockerfile
├── .dockerignore
├── .gitignore
└── app/
    ├── main.py          ← FastAPI app
    └── model.pkl        ← generated by train_model.py (git-ignored)
```

---

## Step 1 — Install dependencies locally

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

---

## Step 2 — Train the model

```bash
python train_model.py
```

Expected output:
```
Loading dataset...
Training RandomForestClassifier...

Test accuracy: 0.967

Classification report:
              precision    recall  f1-score ...
     setosa       1.00      1.00      1.00
 versicolor       1.00      0.92      0.96
  virginica       0.91      1.00      0.95

Model saved to app/model.pkl
```

---

## Step 3 — Test the FastAPI app locally (without Docker)

```bash
cd ml-deploy
uvicorn app.main:app --reload --port 8000
```

Open http://localhost:8000/docs — you get interactive API docs for free.

Test via curl:
```bash
# Health check
curl http://localhost:8000/health

# Predict (setosa sample)
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"data": [5.1, 3.5, 1.4, 0.2]}'
```

Expected response:
```json
{
  "prediction": 0,
  "class_name": "setosa",
  "probabilities": {
    "setosa": 1.0,
    "versicolor": 0.0,
    "virginica": 0.0
  }
}
```

---

## Step 4 — Build the Docker image

```bash
docker build -t ml-model:latest .
```

---

## Step 5 — Run in Docker and test

```bash
# Run container
docker run -p 8000:8000 ml-model:latest

# In a new terminal — same curl commands as above
curl http://localhost:8000/health
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"data": [5.1, 3.5, 1.4, 0.2]}'
```

---

## Checklist before moving to Phase 2

- [ ] `train_model.py` runs and prints accuracy ≥ 0.90
- [ ] `app/model.pkl` exists
- [ ] `uvicorn` serves predictions locally
- [ ] `docker build` completes without errors
- [ ] `docker run` container responds on port 8000
- [ ] `/health` returns `{"status": "ok", "model_loaded": true}`
- [ ] `/predict` returns a class name and probabilities

Once all boxes are checked → move to Phase 2 (GitHub Actions).
